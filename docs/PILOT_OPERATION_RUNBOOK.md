# Mingle-2026 Pilot Operation Runbook

## 목적
- 파일럿 현장에서 운영자/MC가 같은 기준으로 움직이도록 표준 운영 절차를 고정한다.
- 본 문서는 기능 설명이 아니라 실전 운영 행동 지침이다.

## 1) 운영 전 체크리스트
- 관리자 계정 로그인 상태 확인 (`live` 패널 접근 가능 여부).
- 세션 대상 브랜치/세션이 맞는지 확인.
- 테이블 수/테이블 번호 표기(오프라인 표식)와 시스템 값 일치 확인.
- QR 안내물/입장 동선/현장 스태프 위치 확인.
- 비상 연락(총괄/MC/안전 담당) 담당자 확인.
- 운영 시작 전 테스트:
  - 수동 참가자 1명 생성 가능 여부
  - 참가자 이동 버튼 동작 여부
  - ROUND 전환 버튼 노출 여부

## 2) 입장 시작 전 준비
- 세션 상태를 `ROUND_1 시작` 직전 상태로 맞추고, 입장 멘트를 MC와 동기화.
- 앱 미사용자 대응 담당 1명 지정(수동 참가자 생성 전담).
- 테이블 오배치 대응 담당 1명 지정(참가자 이동 전담).
- 신고/차단 판단권자(최종 승인자) 지정.

## 3) 입장 중 운영자 행동
- 앱 사용자: 정상 체크인 유도.
- 앱 미사용자:
  - `수동 참가자 생성`으로 즉시 등록(닉네임/성별/테이블 필수).
  - 등록 후 실제 테이블 착석 여부 재확인.
- 잘못된 테이블 배치 발견 시:
  - 즉시 `참가자 이동` 사용.
  - 이동 후 테이블 상태(인원/경고) 재확인.
- 이상 징후(충돌/불편/반복 민원) 발생 시 운영 로그 기준으로 신고 흐름 안내.

## 4) ROUND_1 시작
- MC 시작 멘트와 동시에 `ROUND_1 시작` 버튼 실행.
- 시작 후 3~5분 내 테이블 상태 확인:
  - `LOW_ACTIVITY` 다수 발생 시 테이블 재배치 준비.
  - `COLLAPSING` 발생 시 즉시 현장 개입(이동/참여 유도).
- 앱 미사용자는 상태 표시에 불리할 수 있으므로 현장 관찰값으로 보정 판단.

## 5) BREAK / 로테이션 준비
- MC 브레이크 멘트와 함께 `BREAK` 전환.
- 브레이크 동안 운영자 작업:
  - 과밀/과소 테이블 정리
  - 지연 입장자 수동 등록 완료
  - 신고 접수 건 1차 정리
- 로테이션(또는 이동) 직전 최종 테이블 밸런스 확인.

## 6) ROUND_2 시작
- MC 안내 후 `ROUND_2 시작` 버튼 실행(Confirm 확인 포함).
- ROUND_2 시작 직후 운영자 점검:
  - 신고/차단 상태 재확인
  - BLOCKED 참가자가 활동 흐름에 끼어들지 않는지 확인
  - 하트 사용 안내 멘트(남은 하트 소진 필요) 공지

## 7) 하트 공개 / 연락처 교환
- 하트 공개는 반드시 운영자가 `하트 공개` 버튼으로 수동 실행(ROUND_2에서만).
- 공개 전 확인:
  - 신고/차단 이슈 긴급건 우선 처리
  - 공개 타이밍을 MC 멘트와 동기화
- 공개 후 정책:
  - 참가자 본인 하트 3개 소진 시에만 보낸 사람 확인 가능
  - 연락처 교환은 상호 하트 + 쌍방 동의 + 최소 1개 연락수단 입력 시만 완료
  - BLOCKED 관련 교환은 진행/유지 불가

## 8) 세션 종료
- MC 종료 멘트 후 `세션 종료` 버튼 실행(Confirm 필수).
- 종료 후 확인:
  - 추가 상태 전환 시도 금지(CLOSED 재오픈 없음)
  - 미처리 신고/운영 이슈 목록화
  - 다음 회차 개선 포인트 기록

## 9) 문제 발생 대응
### 앱 미사용자
- 즉시 수동 참가자 생성으로 운영 객체화.
- 테이블 배치까지 완료해야 처리 종료.

### 잘못된 테이블
- 참가자 이동으로 즉시 수정.
- 반복 오배치 발생 시 현장 안내 동선/표식 점검.

### 신고
- 신고 접수 즉시 대상 분리 원칙.
- 필요 시 즉시 차단 후 사후 검토.

### BLOCKED
- BLOCKED 참가자는 운영 핵심 흐름(공개/교환)에서 제외.
- 관련 연락처 교환은 차단 상태 유지 확인.

### 네트워크 문제
- 서버 상태 복구 전까지 신규 고위험 조작(공개/종료) 보류.
- 오프라인 메모로 임시 운영 기록 후 복구 즉시 반영.
- 중복 클릭/중복 명령은 금지(복구 후 단건 처리).

## 10) 버튼별 사용 기준
### 수동 참가자 생성
- 사용 시점: 앱 미사용자, 현장 즉시 등록 필요 시.
- 금지: 불완전 정보(특히 성별 누락)로 등록 시도.

### 참가자 이동
- 사용 시점: 오배치/테이블 불균형/현장 개입 필요 시.
- 금지: 근거 없는 반복 이동.

### 차단
- 사용 시점: 안전/운영 정책 위반, 즉시 리스크 차단 필요 시.
- 주의: 차단 후 관련 교환/상태 영향 확인 필수.

### ROUND 변경
- 사용 시점: MC 진행 타임라인과 정확히 동기화.
- 금지: 임의/사전 테스트성 전환.

### 하트 공개
- 사용 시점: ROUND_2 운영 안정화 후, MC 공개 멘트 직전.
- 금지: ROUND_2 이전 공개, 중복 공개 시도.
