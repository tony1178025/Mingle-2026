# PILOT Technical Smoke Test

## 목적
- 파일럿 시작 전 개발자가 직접 확인하는 최소 기술 점검표.
- 기능 추가/리팩터링 없이 현재 운영 플로우의 안정성만 확인한다.

## 사용 원칙
- 순서대로 한 번에 수행한다.
- 실패 항목은 즉시 원인 기록 후 재현 절차를 남긴다.
- 성공/실패를 체크박스로 남긴다.

## 체크리스트

### 1) Admin login
- [ ] 관리자 로그인 성공
- [ ] Live 패널 접근 가능
- [ ] 브랜치 권한 범위가 맞게 적용됨

### 2) Session create/open
- [ ] 대상 세션 선택/오픈 상태 확인
- [ ] 세션 기본 정보(브랜치, 테이블 수, 코드) 정상 표시

### 3) QR entry
- [ ] QR 체크인 성공
- [ ] 참가자가 스냅샷에 반영됨
- [ ] 체크인 실패 시 짧은 한국어 에러 표시됨

### 4) Manual participant
- [ ] 수동 참가자 생성 성공 (닉네임/성별/테이블)
- [ ] 생성 직후 목록/카운트에 반영됨
- [ ] 성별 누락 시 거부됨

### 5) Move participant
- [ ] 참가자 이동 성공
- [ ] 이동 이력/상태 반영 확인
- [ ] 동일 테이블 이동 시 거부됨

### 6) Block participant
- [ ] 차단 성공
- [ ] 이미 차단된 대상 재차단 시 거부됨
- [ ] 차단 시 관련 연락처 교환 상태가 BLOCKED로 전파됨

### 7) Table warning
- [ ] LOW_ACTIVITY/COLLAPSING 경고가 테이블 행에 표시됨
- [ ] 상태 변경 후 경고가 즉시 갱신됨

### 8) ROUND_1 → BREAK → ROUND_2
- [ ] ROUND_1 시작 성공
- [ ] BREAK 전환 성공
- [ ] ROUND_2 시작 성공(Confirm 동작 포함)
- [ ] 허용되지 않은 전이는 거부됨

### 9) Trigger reveal
- [ ] ROUND_2에서만 하트 공개 가능
- [ ] 중복 공개 시 거부됨
- [ ] 하트 미소진 참가자는 sender 공개 잠금 유지

### 10) Contact exchange
- [ ] 상호 하트 대상에게만 요청 가능
- [ ] 최소 1개 연락수단 없이 동의 제출 시 거부됨
- [ ] 양측 동의 시 COMPLETED
- [ ] PENDING/BLOCKED 상태에서 연락처 필드 노출되지 않음

### 11) CLOSED 이후 명령 차단
- [ ] 세션 종료(CLOSED) 후 admin 명령 전체 차단 확인
- [ ] 에러 메시지가 사용자에게 명확히 표시됨

### 12) CAS stale version error
- [ ] 오래된 version으로 admin 명령 전송 시 거부됨
- [ ] 메시지: "세션이 갱신되었습니다. 다시 시도해 주세요."
- [ ] toast로 운영자에게 명확히 표시됨

### 13) Full test command
- [ ] 최소 단위 테스트 실행
- [ ] 파일럿 전 전체 테스트 1회 실행
- [ ] 실패 테스트가 있으면 파일럿 진행 보류

권장 실행:
- `npm test`

## 종료 기준
- 필수 항목(1~12) 모두 통과
- 전체 테스트 실행 기록 남김
- 실패 항목 0건 또는 우회 불가 이슈 없음
