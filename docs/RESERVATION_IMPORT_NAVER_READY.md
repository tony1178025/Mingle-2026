# Reservation Import Naver-Ready

## 목적

네이버 API 연동 없이도, 네이버 예약 관리자에서 내려받은 엑셀/CSV를 Level 1 운영에 즉시 사용할 수 있게 한다.

## Level 1 원칙

- 네이버 직접 API 연동은 후순위
- 네이버 관리자 엑셀 다운로드 + 수동 CSV 업로드를 1차 공식 대응으로 사용
- 예약 데이터는 참가자 자동 생성이 아니라 **입장 자격 검증** 용도

## 권장 절차

1. 네이버 예약/주문 관리자에서 예약 목록 다운로드
2. CSV로 정리 또는 바로 CSV 변환
3. Admin `예약 현황`에서 업로드
4. 미리보기/검증/중복 전화 확인
5. 현장 운영에 반영

## 지원 컬럼 별칭

- 이름: `이름`, `예약자명`, `방문자명`
- 전화번호: `전화번호`, `휴대폰`, `연락처`
- 예약일: `예약일`, `이용일`, `방문일`
- 예약시간: `예약시간`, `이용시간`
- 상품명: `상품명`, `예약상품`, `프로그램명`
- 회차: `회차`, `옵션`, `상품옵션`
- 결제상태: `결제상태`
- 예약상태: `예약상태`
- 메모: `요청사항`, `메모`
- 성별: `성별`
- 출생연도/나이: `출생연도`, `나이`

## 표준 정규화 필드

- `source` (`NAVER` | `MANUAL` | `IMWEB` | `CSV`)
- `externalReservationId`
- `branchId`
- `eventDate`
- `slot` (`1부` | `2부` | `1+2부`)
- `name`
- `phone`
- `normalizedPhone`
- `gender`
- `birthYear` / `age`
- `paymentStatus`
- `reservationStatus`
- `memo`
- `rawRow`
- `importedAt`

## 개인정보/마스킹 주의

- 네이버 원본 데이터는 보존기간/활용 범위를 별도 정책으로 관리
- 이용완료 이후 고객 데이터 활용은 동의/정책 기준으로 제한
- 장기 CRM 복사는 무조건 수행하지 않음 (동의/정책 기반으로만)
